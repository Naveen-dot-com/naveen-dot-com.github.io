4. How Requests Are Routed (Cloudflare Worker Proxy)

To protect the Google Gemini API key from being exposed in client-side code, all AI requests are routed through a Cloudflare Worker — a lightweight serverless proxy deployed on Cloudflare's edge network.

  • The Worker receives your message text and forwards it to the Google Gemini API
  • The Worker does not log, store, or retain any request or response content
  • The Gemini API key is stored as an encrypted Cloudflare secret and is never visible to the client
  • The Worker only handles CORS and request forwarding — no data processing or analytics occur

Cloudflare's data handling is governed by the Cloudflare Privacy Policy. Cloudflare may process network-level metadata (e.g. IP addresses) as part of standard routing infrastructure, but we have no access to this data.

5. Conversation Data & Local Storage

Your conversation history is saved only on your own device using your browser's local storage (localStorage). It is never transmitted to us or any server we operate.

The following data is stored locally on your device only:

  • Your chat history and past conversation sessions
  • Your language preference (e.g. English, Hindi)
  • Your theme preference (light or dark mode)
  • Whether you have dismissed the notification permission prompt

You can delete all locally stored data at any time by clearing your browser or app data via Android Settings → Apps → Oshowani → Storage → Clear Data.

6. Push Notifications (Coming soon)

Oshowani offers optional daily push notifications containing a short spiritual thought related to meditation, mindfulness, or awareness. These notification messages are generated using the Google Gemini API via our Cloudflare Worker proxy.

  • Permission is always optional. The app will ask for your permission before sending any notifications. You can decline or revoke permission at any time in your device Settings → Notifications → Oshowani.
  • Notifications are generated by calling the Gemini API once per day — no dedicated push notification server or user subscription endpoint is used
  • On Android, notifications are displayed via the Android system notification service using the Web Push / Service Worker API
  • No device push token, subscription ID, or user identifier is collected or stored by us
  • Periodic Background Sync may be used on supported devices to trigger daily notification generation. This makes one standard Gemini API call and uses no additional personal data
  • If the Gemini API is unavailable, a pre-written offline fallback message from a local pool is used — no network request is made in that case

If you do not grant notification permission, no notification-related network requests are ever made.

Oshowani uses the following third-party services. No other SDKs, libraries, analytics tools, advertising networks, or trackers are included in the app.

10. Data Security

All communication between your device and third-party services uses HTTPS with TLS 1.2 or higher. The Gemini API key is stored as an encrypted server-side secret within Cloudflare and is never transmitted to or accessible by the client app. Since we do not operate any database or backend server that stores your data, there is no risk of a server-side breach affecting your conversations or personal information.

11. Data Retention & Deletion

We do not retain any user data on our servers. All data retention is on your own device only:

  • Chat history — stored in your device's local storage until you clear it
  • Preferences — stored locally until you clear app data
  • Notification dismissal flag — stored locally until you clear app data

To delete all Oshowani data from your device, go to Android Settings → Apps → Oshowani → Storage → Clear Data. This action is immediate and permanent. There is no server-side data to request deletion of, as we hold none.
